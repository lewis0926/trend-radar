Trend Radar is an assistant app that helps identify which industries are trending based on past price action.

## Project
Send weekly email notifications to users tracking a list of ETFs/indices with exposure to all major market industries — tech, healthcare, finance, energy, consumer discretionary, and more.

## Tech Stack
- Python 3.10+
- venvs for dependency management
- Resend for email notifications
- Pkl for config management — generate `config.json` from a Pkl file and parse it in the app

## Config Structure
- `config/schema/Config.pkl` — typed schema with defaults. All fields are defined here. As config grows, define additional types/modules and import them here.
- `config/<env>.pkl` — environment-specific file (e.g. `dev.pkl`). Amends the schema and supplies env-specific values like API keys.
- `config.json` — generated by running `pkl eval "config/<env>.pkl" -f json -o "config.json"`. This is what the app reads.

## Running
```bash
python run.py
```

## Adding a New ETF
1. Add the typed entry to `config/schema/Config.pkl`
2. Override or extend in `config/<env>.pkl` if needed
3. Regenerate: `pkl eval "config/<env>.pkl" -f json -o "config.json"`

## Scheduling
Currently run manually. Intended to be triggered weekly (e.g. via cron or GitHub Actions) — no scheduler is set up yet.

## Coding Standards
- `config.json` is the single config file. All configuration — including secrets like API keys — goes in env Pkl files and flows into `config.json` (which is gitignored). No `.env` files, no environment variables.
